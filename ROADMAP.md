# Roadmap

## Phase 0: Validation

Status: active.

- Browser demo.
- Mock-safe proxy.
- Public launch copy.
- Manual validation metrics.

## Phase 1: Local Proxy MVP

- OpenAI-compatible proxy.
- Request logs.
- Cost estimates.
- Route rules.
- Budget caps.
- Private-path warnings.
- Local dashboard.

Current status:
- Implemented in `modelbudget-mvp/`.

## Phase 2: Provider Support

- Ollama forwarding.
- OpenAI-compatible hosted forwarding behind explicit opt-in.
- Streaming responses.
- Per-route provider config.
- Config validation for missing models/API keys.

## Phase 3: Developer Integrations

Target integrations to validate:
- Continue.
- Open WebUI.
- OpenClaw.
- Hermes.
- Codex/OpenAI-compatible tools.
- Cursor where configurable.
- Claude Code where configurable.

## Phase 4: Team Features

- Shared policy files.
- Per-user/repo/project budgets.
- Team dashboard.
- Historical reports.
- Privacy policy enforcement.
- Exportable spend reports.

## Phase 5: Monetization

Possible paid features:
- Team dashboard.
- Hosted monitoring.
- Shared policies.
- Managed setup.
- Compliance/privacy reports.

No monetization should be added until developer demand is validated.
