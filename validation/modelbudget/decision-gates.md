# ModelBudget Decision Gates

## Validation Window

Run a 7-14 day validation window after public launch.

## Continue Criteria

Continue to build the proxy MVP if at least two of these happen:
- 300+ visitors to the calculator/demo.
- 50+ calculator runs.
- 25+ GitHub stars or equivalent saves/bookmarks.
- 10+ meaningful comments from developers.
- 5+ requests for a working proxy.
- 2+ people ask about team dashboards, privacy rules, or budget enforcement.

## Strong Continue Criteria

Prioritize this product if:
- Someone shares an actual AI bill or agent cost problem.
- A team asks about using it across multiple developers.
- Users ask for integration with Cursor, Claude Code, Codex, OpenClaw, Hermes, Continue, or Open WebUI.

## Pivot Criteria

Pivot if:
- Developers say existing tools already solve it well.
- Feedback focuses only on local model setup, not spend visibility.
- Users like the calculator but do not want a proxy/gateway.
- No one asks for budget enforcement, privacy rules, or team reporting.

## Next MVP If Continued

Build:
- Local OpenAI-compatible proxy.
- Request logging.
- Cost estimator.
- Simple route rules.
- Budget cap.
- Redaction/private-path warning.
