# Feedback Questions

Use these questions when sharing ModelBudget with developers.

## Usage

- Which AI coding tools do you use?
- Do you know what an average agent run costs?
- Have you ever hit usage limits or had surprise AI costs?
- Do you use local models today?
- Do you use OpenRouter or multiple model providers?

## Pain

- What is most painful: cost, privacy, latency, model choice, or visibility?
- Do you need budget caps per repo/project/user?
- Would you trust a local proxy between your agent and model provider?
- What would make you not install this?

## Feature Demand

- Which feature matters most?
  - Cost logging.
  - Budget caps.
  - Local model fallback.
  - Private-path blocking.
  - Team dashboard.
  - Provider routing.
  - Exportable reports.

## Buying Intent

- Would you pay for this personally?
- Would your team pay for this?
- What price would feel reasonable?
- What would need to exist before paying?

## Strong Signal Notes

Strong validation comments sound like:
- "Can I use this with X?"
- "Can it enforce a budget?"
- "Can it block secrets?"
- "Can I run this for my team?"
- "Can it show cost by repo?"
- "Can it support OpenRouter/Ollama?"
